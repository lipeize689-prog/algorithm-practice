# 🔥 LFU Cache（带过期机制）

## 🧩 Problem Description

实现一个带有容量限制的 **LFU（Least Frequently Used）缓存**。

缓存支持两种操作：

### 1️⃣ `get key`

* 如果 `key` 存在，返回对应的 `value`
* 如果不存在，返回 `-1`

---

### 2️⃣ `put key value`

* 如果 `key` 已存在：更新其 `value`
* 如果不存在：插入该键值对
* 如果缓存已满：

  * 删除 **访问频率最低（LFU）** 的键
  * 若存在多个频率相同：

    * 删除 **最久未使用（LRU）** 的键

---

## ⏱️ 额外要求（重点）

每次操作前，都需要进行一次检查：

> ❗ 删除所有“超过 T 时间单位未被访问”的键

说明：

* 每个操作（get / put）都会消耗 1 个时间单位
* 如果某个 key 在最近 T 次操作中都没有被访问（get 或 put），则必须被移除

---

## ⚡ 时间复杂度要求

* `get`：O(1)
* `put`：O(1)

---

## 📥 Input Format

第一行输入三个整数：

```
N M T
```

* `N`：缓存容量
* `M`：操作次数
* `T`：过期时间阈值

接下来 M 行，每行表示一个操作：

* `op = 1`：get 操作

  ```
  1 k
  ```

* `op = 2`：put 操作

  ```
  2 k v
  ```

---

## 📤 Output Format

### 1️⃣ 对每个 get 操作输出一行：

* 存在：输出 value
* 不存在：输出 -1

---

### 2️⃣ 最后输出缓存中的所有值：

* 按 **最后更新时间（从旧到新）排序**
* 如果缓存为空，输出：

```
empty
```

## 🚧 难点分析

这道题的核心难点在于：

### 🔥 1. LFU + LRU 双重策略

* 优先删除频率最低
* 再按时间淘汰（LRU）

---

### 🔥 2. 过期机制（T 时间限制）

* 每次操作前都要检查
* 需要高效删除“过期节点”

---

### 🔥 3. O(1) 复杂度要求

通常需要以下结构组合：

* `HashMap`：key → 节点
* `freq → 双向链表`
* 记录：

  * 访问频率
  * 最近访问时间

---

## 🧠 关键思路提示

* 用 **频率分层结构（LFU标准做法）**
* 每个频率维护一个 **LRU链表**
* 用时间戳 + 懒删除 或 队列 处理过期

---

## 📊 数据范围

```
1 ≤ N, T ≤ 2 × 10^4
1 ≤ M ≤ 10^6
1 ≤ k ≤ 3 × 10^8
0 ≤ v ≤ 10^9
```
